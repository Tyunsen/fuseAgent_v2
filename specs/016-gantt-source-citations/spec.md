# Feature Specification: Gantt Source Citations

**Feature Branch**: `[016-gantt-source-citations]`  
**Created**: 2026-04-04  
**Status**: Draft  
**Input**: User description: "1.时间甘特图需要按真实事件和真实时间展开，不能全部挤在一天，也不能显示时间结论1/2这种错误标签；2.时间模式主图下方不要再出现额外的关联卡片；3.参考文档来源入口放到点赞复制同一操作区，点击后从右侧抽屉展示来源，正文里需要论文式引用标号如[1]。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 时间脉络主图纠偏 (Priority: P1)

作为在 QA 页面使用时间脉络模式的用户，我希望主图直接展示按时间排序的关键事件甘特图，而不是把所有事件挤在同一天，也不是显示“时间结论 1/2/3”这类内部编号，这样我才能一眼看懂 3 月的重要事件是如何展开的。

**Why this priority**: 时间脉络模式的主要价值就是“按时间整理”。如果主图时间定位错误或标签不可读，这个模式本身就失效。

**Independent Test**: 在一个包含多个 3 月事件的知识库中发起时间脉络问题，确认主图是甘特图、标签是事件名、时间位置能区分不同日期或时间段。

**Acceptance Scenarios**:

1. **Given** 时间脉络回答引用了多个具有不同日期或时间段的事实，**When** 回答渲染完成，**Then** 主图必须显示为时间甘特图，并把这些事实展开到各自对应的时间位置，而不是全部落在同一天。
2. **Given** 时间脉络回答已经生成主图，**When** 用户查看图中的任务标签，**Then** 标签必须直接显示事件名称或事件短语，而不能显示“时间结论 1”“时间结论 2”之类的内部标题。
3. **Given** 时间脉络回答已经生成主图，**When** 用户查看主图区域下方，**Then** 页面不得再出现额外的次级时间卡片、实体筹码卡片或重复的辅助子图卡片。

---

### User Story 2 - 参考来源抽屉 (Priority: P1)

作为阅读 QA 回答的用户，我希望“参考文档来源”入口放在点赞、复制等消息操作区旁边，并在点击后从右侧打开抽屉查看来源，这样我可以在不打断阅读主内容的情况下快速查看证据。

**Why this priority**: 来源查看是每条回答的高频操作。入口位置和交互方式直接决定可发现性和阅读流畅度。

**Independent Test**: 在任一带引用的回答中查看消息操作区，确认存在“参考文档来源”入口；点击后右侧出现抽屉，展示来源列表和片段。

**Acceptance Scenarios**:

1. **Given** 一条回答存在引用来源，**When** 该回答渲染完成，**Then** 操作区必须出现“参考文档来源”入口，并与点赞、复制等操作处于同一排。
2. **Given** 用户点击“参考文档来源”入口，**When** 交互触发后，**Then** 页面必须从右侧打开来源抽屉，并展示本次回答使用的来源列表、位置标签与原文片段。
3. **Given** 来源抽屉已经可用，**When** 用户查看回答主内容下方，**Then** 原来的底部折叠来源卡片不得再继续显示。

---

### User Story 3 - 正文引用标号 (Priority: P2)

作为需要核验结论的用户，我希望回答正文中的具体结论后面直接出现 `[1]`、`[2]` 这种论文式引用标号，并且这些编号能和右侧来源抽屉中的条目对应起来，这样我能立即知道每一句结论来自哪里。

**Why this priority**: 只有来源列表没有正文引用编号，用户仍然需要自己猜测“哪句话对应哪条来源”，可核验性不足。

**Independent Test**: 针对一条带引用的回答，检查正文是否包含 `[n]` 标号，并确认这些编号在来源抽屉中存在对应条目。

**Acceptance Scenarios**:

1. **Given** 回答正文包含来自知识库的结论，**When** 回答渲染完成，**Then** 每条被引用的结论后都应显示可见的 `[n]` 引用标号。
2. **Given** 回答正文中出现 `[n]` 标号，**When** 用户对照右侧来源抽屉查看来源，**Then** 抽屉中必须存在相同编号对应的来源条目。
3. **Given** 一条回答没有任何可用引用来源，**When** 回答渲染完成，**Then** 页面不得伪造 `[n]` 标号，也不得展示误导性的来源入口。

---

### Edge Cases

- 当时间证据只精确到“某月”或“某时间段”时，系统如何避免伪造具体到某一天的甘特图位置？
- 当多个事件发生在同一天时，系统如何在同一日期内仍保持事件标签可读，而不是互相覆盖或退化成内部编号？
- 当回答正文引用了多条来源，但其中部分来源没有精确段落定位时，右侧来源抽屉如何保持可读且不误导用户？
- 当用户打开来源抽屉后再次切换回答或切换模式时，系统如何避免继续显示上一条回答的来源内容？
- 当回答正文已经包含裸露的 `graph TD` 或其他 Mermaid 文本时，系统如何避免这些文本污染最终展示？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须让时间脉络模式的主图始终以时间甘特图形式展示本次回答的时间线结果。
- **FR-002**: 系统必须优先使用回答中可绑定到证据的真实事件名称、事件短语或结论短语作为甘特图任务标签，而不是使用“时间结论 1/2/3”之类的内部编号。
- **FR-003**: 系统必须根据已归一化的时间信息，把不同事件放置到各自对应的日期、月份或时间段位置，不能在存在多个不同时间点时把所有事件挤压到同一天。
- **FR-004**: 当证据只提供月级或区间级时间时，系统必须保留该级别的时间粒度，并不得伪造不存在的具体日期。
- **FR-005**: 时间脉络模式必须只保留一个主图区域，不得在主图下方继续展示额外的关联卡片、时间筹码卡片或重复子图卡片。
- **FR-006**: 对于存在引用来源的回答，系统必须在消息操作区提供“参考文档来源”入口，并与点赞、复制等操作处于同一操作带。
- **FR-007**: 用户点击“参考文档来源”入口后，系统必须从页面右侧打开来源抽屉展示来源列表，而不是在回答正文下方展开折叠卡片。
- **FR-008**: 右侧来源抽屉必须展示来源编号、文档名称、定位信息、摘要片段和原文展开能力。
- **FR-009**: 系统必须在回答正文中为每条被引用的结论显示可见的 `[n]` 编号，并让这些编号与来源抽屉中的条目编号一一对应。
- **FR-010**: 同一条来源可以被多处正文结论复用，但系统必须保持编号稳定，不得为同一来源重复分配不同编号。
- **FR-011**: 当回答没有引用来源时，系统不得显示“参考文档来源”入口，也不得在正文中显示伪造的 `[n]` 编号。
- **FR-012**: 当回答正文包含 Mermaid 代码块或裸露的 Mermaid 文本时，系统必须避免这些图形源码直接污染用户可见正文。
- **FR-013**: 以上展示规则必须适用于新生成的回答和重新打开历史会话后重新渲染的回答，只要该回答仍保留结构化引用数据。

### Key Entities *(include if feature involves data)*

- **时间脉络事件项**: 时间脉络主图中的单个事件单元，包含事件标签、时间定位和关联来源。
- **正文引用标号**: 显示在回答正文中的 `[n]` 标记，用于把具体结论映射到来源条目。
- **来源抽屉**: 从消息操作区打开的右侧面板，承载本次回答的来源列表与原文片段。
- **来源条目**: 来源抽屉中的单条文档证据，包含编号、文档名称、位置、摘要与原文。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在 `E:\codes\fuseAgent_v2\iw_docs` 验收数据集上，针对时间脉络问题“3月发生了什么？”，主图必须显示为甘特图，并且可见任务标签中 0 条允许使用“时间结论 1/2/3”之类的内部编号。
- **SC-002**: 在同一验收问题下，只要引用证据中存在 3 个及以上不同的时间点或时间段，甘特图中也必须出现至少 3 个不同的时间落点或时间区段，而不是全部挤在同一天。
- **SC-003**: 在任一带引用的回答中，消息操作区必须出现单一“参考文档来源”入口；点击后右侧抽屉必须在 1 秒内可见，且回答正文下方不再出现旧的折叠来源卡片。
- **SC-004**: 在任一带引用的回答中，正文必须出现至少 1 个 `[n]` 引用标号，并且抽屉中每个被展示的来源编号都能在正文中找到对应编号。
- **SC-005**: 按县章系统级验收要求，使用 `iw_docs` 全量导入后，向量、全文与图索引总耗时仍须不超过 4 分钟，且知识图谱页面节点数大于 80、关系数大于 100。

## Assumptions

- 时间脉络主图仍基于现有的回答证据、时间归一化结果和图支持数据生成，不引入新的外部数据源。
- 正文引用标号只针对当前回答实际使用的结构化来源生成，不要求为没有结构化来源的数据强行补号。
- “参考文档来源”入口是消息级操作，不需要升级为全局页面级工具栏入口。
- 若历史回答仍保留结构化引用与模式信息，则前端重新渲染时应遵守新展示规则；不要求回写历史消息存量文本。
